# VitaPick Frontend

VitaPick은 건강기능식품 쇼핑몰과 관리자 페이지를 함께 구현한 팀 프로젝트입니다.  
이 저장소는 VitaPick의 Frontend 저장소이며, 저는 관리자 페이지 화면 구현과 Spring Boot Backend API 연동을 담당했습니다.

관리자 페이지는 운영자가 회원, 상품, 주문, 리뷰, 고객센터 데이터를 확인하고 처리할 수 있는 화면을 목표로 구현했습니다.  
대시보드에서는 서비스 운영 상태를 한눈에 볼 수 있도록 매출, 주문, 회원, 문의 데이터를 시각화했고, 각 관리 화면에서는 목록 조회에서 상세 확인, 수정, 답변 처리까지 이어지는 흐름을 구성했습니다.

## 핵심 구현 내용

### 1. 관리자 대시보드

관리자 대시보드는 운영자가 서비스 상태를 빠르게 확인할 수 있는 화면으로 구현했습니다.

Backend에서 집계한 데이터를 Axios로 받아오고, Frontend에서는 카드, 그래프, 목록 형태로 화면에 표시했습니다.  
오늘 매출, 오늘 주문 수, 이번 달 매출, 월별 회원 수, 월별 주문 수, 상품별 매출 TOP5, 문의 처리 현황을 한 화면에서 확인할 수 있도록 구성했습니다.

```text
Spring Boot 관리자 API 호출
→ 매출, 주문, 회원, 문의 데이터 응답
→ React 화면에서 카드·그래프·목록 형태로 표시
```

<br />

### 2. 관리자 접근 제어

관리자 페이지 접근은 Frontend에 저장된 role 값만으로 처리하지 않았습니다.

처음에는 화면에서 저장된 권한 값을 기준으로 관리자 접근을 판단할 수 있었지만, sessionStorage 값은 사용자가 직접 수정할 수 있습니다.  
그래서 관리자 페이지 진입 시 Backend의 관리자 권한 확인 API를 호출하고, 검증 결과에 따라 접근 여부를 결정하도록 수정했습니다.

```text
관리자 URL 접근
→ Backend 관리자 권한 확인 API 호출
→ ADMIN 권한 확인 성공 시 관리자 페이지 유지
→ 권한 없음 또는 API 호출 실패 시 메인 화면으로 이동
```

이 흐름을 통해 관리자 접근 판단 기준을 Frontend 상태값이 아니라 Backend 검증 결과로 옮겼습니다.

<br />

### 3. 관리자 주요 화면

관리자 화면은 운영자가 실제로 처리해야 하는 업무 흐름을 기준으로 나눴습니다.

회원 관리에서는 회원 기본 정보와 주문 내역을 확인할 수 있게 했고, 상품 관리에서는 상품 상세 조회와 수정 흐름을 연결했습니다.  
주문 관리는 결제완료 주문을 기준으로 조회하도록 구성했고, 리뷰 관리에서는 리뷰 확인과 관리자 답글 등록·삭제 기능을 연동했습니다.  
고객센터 관리에서는 공지글, FAQ 항목, 1:1 문의 답변 처리를 한 화면 흐름 안에서 관리할 수 있도록 구현했습니다.

<br />

### 4. 엑셀 리포트 다운로드 연동

대시보드에서는 관리자 운영 데이터를 엑셀 파일로 내려받을 수 있도록 다운로드 API와 연동했습니다.

Frontend에서는 다운로드 버튼 클릭 시 Backend의 엑셀 다운로드 API를 호출하고, 응답으로 받은 파일 데이터를 브라우저에서 다운로드 처리했습니다.  
관리자가 화면에서 보는 운영 데이터를 파일로도 확인할 수 있도록 연결한 기능입니다.

```text
엑셀 다운로드 버튼 클릭
→ Backend 엑셀 다운로드 API 호출
→ Excel 파일 응답 수신
→ 브라우저에서 파일 다운로드 처리
```

## 배포 경험

Frontend는 AWS EC2 서버에서 Docker와 Nginx를 사용해 배포했습니다.

React 빌드 결과물은 Nginx를 통해 정적 파일로 서빙했고, Backend API 요청은 Nginx 설정을 통해 Spring Boot 서버와 연결했습니다.  
GitHub Actions를 사용해 코드 변경 후 EC2 서버에서 재빌드와 재실행이 이루어지도록 배포 흐름을 구성했습니다.

배포 과정에서는 Nginx의 `proxy_pass` 설정 오류가 발생했고, 요청 경로와 location 설정을 수정해 문제를 해결했습니다.

<br />

## 구현하면서 중점적으로 본 부분

관리자 페이지는 화면을 많이 만드는 것보다 데이터 흐름을 정확히 연결하는 것이 중요했습니다.

Frontend에서는 사용자가 보는 화면과 API 요청 흐름을 담당하고, 권한 검증이나 매출 집계처럼 조작되면 안 되는 처리는 Backend 응답을 기준으로 화면에 반영했습니다.  
특히 관리자 접근 제어는 Frontend 값만 믿지 않고 Backend 확인 API를 거치도록 수정하면서, 화면 구현과 보안 흐름을 함께 점검할 수 있었습니다.
